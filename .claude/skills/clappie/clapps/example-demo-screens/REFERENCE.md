# Example Demo Screens Reference

Example views for learning display-engine patterns. Push any of these to see how components, layouts, and interactions work.

## Screens

| Screen | What it demonstrates |
|--------|---------------------|
| `hello-world` | Simplest possible view — title, description, ctx.draw(), keyboard shortcuts, scrolling |
| `all-components` | Every UI kit component: inputs, selects, toggles, checkboxes, radios, buttons, alerts, progress, loader |
| `layout-demo` | Layout configurations and positioning |
| `easy-stack-demo` | Stack layout patterns |
| `stack-test` | Stack layout edge cases |
| `full-width-inputs` | Input field sizing and width behavior |
| `input` | Text input variations |
| `textarea` | Multi-line input examples |
| `submit-data-types` | Different ctx.submit() data formats |
| `status-bar-mockup` | Status bar UI pattern |
| `incorrect-with-error` | Error state handling |

## Usage

```bash
clappie display push example-demo-screens/hello-world
clappie display push example-demo-screens/all-components
clappie display push example-demo-screens/layout-demo
```

## Start Here

- **New to displays?** Push `hello-world` — shows the minimal view lifecycle (create, init, render, onKey).
- **Building a form?** Push `all-components` — shows every UI kit component with working interactions.
- **Layout issues?** Push `layout-demo` or `easy-stack-demo` — shows how stacking and width work.
