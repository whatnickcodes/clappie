Clean up stale and orphaned sidekicks.

Steps:
1. Read the port from recall/settings/sidekick/port.txt (plain integer, default 7777 if missing)
2. Check if server is running: GET http://localhost:{port}/health
   If it fails, skip this check entirely.
3. GET http://localhost:{port}/status to fetch all sidekicks as JSON
4. For each sidekick where status === "active":
   a. Extract paneId from the sidekick object
   b. Check if pane is alive:
      - Run: `tmux list-panes -a -F "#{pane_id}" | grep -q "{paneId}"`
      - If grep returns false, pane is dead → mark sidekick as stale
   c. Check lastActivity timestamp:
      - Parse ISO datetime from lastActivity (or startedAt if missing)
      - If older than 30 minutes from now, mark as stale
5. For each stale sidekick:
   - Kill the pane if alive: `tmux kill-pane -t {paneId} 2>/dev/null`
   - POST to http://localhost:{port}/abort
     Body: {"sidekickId": "{sidekick-id}"}
6. Log results to recall/logs/heartbeat/YYYY-MM-DD.txt (APPEND to daily file):
   Format: [HH:MM] sidekick-cleanup ✓ active: A completed: C stale: S (cleaned)
   Example: [14:30] sidekick-cleanup ✓ active: 2, completed: 15, stale: 0

   NEVER create standalone files like sidekicks-cleanup.log or sidekick-cleanup-DATE.txt.
   ALWAYS append ONE line to the daily heartbeat file. That's it.

Report in log:
- Total active sidekicks found
- Total completed sidekicks
- Count of stale sidekicks found and cleaned
- Any errors (server unreachable, curl failures, etc.)

Agent tips:
- Use simple bash for date math or use `date` command to check timestamp age
- Curl examples:
  curl -s http://localhost:7777/status | jq '.sidekicks[] | select(.status=="active")'
  curl -s -X POST http://localhost:7777/abort -H "Content-Type: application/json" -d '{"sidekickId":"id-here"}'
- Be defensive: handle missing paneId gracefully (shouldn't happen, but log if seen)
